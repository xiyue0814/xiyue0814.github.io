了解网页Icon的用法

发表于 2018-07-18 | 分类于 UI相关

前言

在如今的Web页面中，随便打开一个网站，其中图片所占的比例是非常高的。这些图片不仅包含banner或者产品图，还有用于提示作用的小图标等。当然啦，有了这些图片的引导，页面的可读性和美观性都有所提升。可是每加载一个图片，都会产生一次http请求，影响网站的响应速度。
为了减少http的请求次数，前端er总结出许多方法，从最初的css雪碧图 -> css3制作图标 -> font字体 -> svg矢量图，这些方法可以在不同的场景下使用。下面，我们就来一一介绍吧~

CSS雪碧图

原理：把许多小图标合成一张大图，在使用时，通过background-position属性来精确获取具体的图标位置。这样的好处是把多个请求合并成一个。缺点是这种方法不适用banner图或者产品图等大图。

这儿有个很好的在线工具，很方便制作雪碧图，而且背景位置都给你标出来了，直接copy就好。

font字体

随着各种高清网膜屏幕的出现，现在web设计也需要考虑各种高清屏幕的显示效果.所以前端在代码实现的时候，也需要根据不同的屏幕来输出不同分辨率的图片.在开发过程中就需要准备两套不同尺寸的图片来应对.一套用在普通屏幕上，一套则用在高清屏幕上。这样就增加了开发的难度及降低开发效率。
为了解决这种情况,font icon矢量图出现了.它的优势是可以随意改变大小，而不影响图片质量。
目前有两个常用的字体库，一个是Font Awesome它可以与bootstrap很好的结合起用。另一个是iconfont,它是阿里的一个字体库，上面有很多图片，你可以根据自己的需求生成字体文件。字体图标的引用方式有三种:font-class引用、unicode引用.这两种引用方式是比较常用的。还有一种symbol引用,这是最新的一种使用方式，应该会变成主流方式,目前仅兼容ie9以上的浏览器。
