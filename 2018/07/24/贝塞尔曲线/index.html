让页面动起来

发表于 2018-07-24 | 分类于 UI相关

transition动画

随着css3的广泛使用，或多或少都会用到transition的属性transition-timing-function(根据时间的推进改变属性值的变化速率)来做动画效果。而这个属性本身带有6个值可供选择：

  1. ease –(逐渐变慢) 默认值
  2. linear –(匀速)
  3. ease-in –(加速)
  4. ease-out –(减速)
  5. ease-in-out –(加速然后减速)
  6. cubic-bezier --(自定义时间曲线)

上面前5个都有相应的贝塞尔曲线，如ease对应的贝塞尔曲线是cubic-bezier(0.25, 0.1, 0.25, 1.0)。然而，正是因为有5个现成的效果，导致cubic-bezier经常被忽视。这里主要了解怎么用cubic-bezier。

怎样使用贝塞尔曲线？

贝塞尔曲线之所以被忽视，主要是它不直观。
cubic-bezier(P1x, P1y, P2x, P2y)
上面的四个参数，其实是贝塞尔曲线中P1和P2两个点的相应的x轴和y轴值。

因为在css3中的贝塞尔曲线P0和P3是固定了的。P0点的值总是(0, 0)，P3点的值总是(1, 1)。所以P1和P2两个点的值总是在[0, 1]区域内，否则无效。
其中x轴是动画所花的时间，y轴是被更改的属性
http://cubic-bezier.com网站是个很好的工具，可以在线调试贝塞尔曲线，当满足你想要的运动效果时，copy过来直接用就好。
如果你自己怎么调都觉得不满意时，可以去缓动函数速查表网站选择常用的运动效果，也是很方便呢。

requestAnimationFrame函数

假如，我是说假如你的项目要运行到某些不那么现代化的浏览器上。<咳咳，IE往哪里躲呢？> 它并不支持css3的transition以及transform属性，我们该怎么办呢？
是的。我们可以选择setTimeout或setInterval来不断更新元素的状态位置等实现动画。目前又多了一种实现动画的方案，即requestAnimationFrame()方法。它类似于setTimeout递归调用同一方法不断更新画面，以达到动起来的效果。但它是由浏览器专门为动画提供的API。在运行时，浏览器会自动的对它进行优化。并且页面不是在激活状态下，动画会自动暂停，有效节省了CPU的开销。很友善。
它使用也很简单

注意到它并不需要你手动设置时间间隔了吗，那它要怎样确认该隔多少时间调用一次回调函数呢？
在Javascript中，setTimeout任务被放进了异步队列中，只有等主线程上的任务被执行完了后，才会执行异步队列中的任务，因此会导致setTimeout的实际执行时间一般比设置的时间要晚。另外不同设备的屏幕刷新率(就是屏幕每秒刷新多少次，大部分都是60次，就是每16.7ms刷新一次)。而setTimeout只能设置一个固定的时间值，这个时间不一定和屏幕的刷新时间相同。基于这两种原因，导致了setTimeout的执行步调和屏幕的刷新频率不一致，从而引起丢帧的现象，引起动画卡顿。
明白requestAnimationFrame不设定间隔时间的用意了吧。这也是它最大的优势：由系统来决定回调函数执行的时间。具体来说，如果屏幕刷新率是60Hz，那么我的间隔时间就是16.7ms，如果屏幕刷新率是75Hz，那么我的间隔时间就是13.3ms，它能保证回调函数在屏幕的每次刷新过程中，只被执行一次，这样就不会出现丢帧，动画卡顿的现象出现了。
