函数节流和函数防抖的区别

发表于 2018-08-03 | 分类于 JS相关

##函数节流&函数防抖
函数节流和函数防抖在一般取名为throttle和debounce，主要用在js性能优化上，尤其是对于函数被触发的频率太高，如scroll、mousemove、resize事件。如果处理事件的函数稍微复杂或者大量的操作DOM，响应速度跟不上触发频率，就会导致卡顿、延迟或者假死现象。实际上在很多情况下并不需要如此夸张的调用执行函数。
throttle和debounce概念很类似，并且都是利用setTimeout将被执行的函数延迟一段时间执行。但是有细微的差别。我们先看看定义：

函数节流：定义一个wait等待时间，执行函数在wait秒内最多只能执行一次。

函数防抖：定义一个wait等待时间，执行函数在触发事件结束后，等待wait秒执行一次，如果没有到wait秒，又有新的事件进来，则重新计算时间。

什么是设计模式？

设计模式其实是一些经过了大量实际项目验证的优秀解决方案。它是解决某些问题的一种思想，与具体的使用语言无关。
从某些角度来讲，设计模式可能带来代码量的增加，或许也会把系统的逻辑搞的更复杂。但软件的开发成本不仅仅在开发阶段，更包含维护阶段。而设计模式能在维护阶段为我们带来好处。设计模式的实现都遵循一条原则，即‘找出程序中变化的地方，并将其封装起来’。
学习模式如果脱离了实际的场景，都是无意义的。因为很多模式的类图看起来几乎是一样的。所以在学习某种设计模式时一定要清楚：这个模式是为了解决什么问题而出现的。

发布-订阅者模式

发布-订阅者模式是指一个对象维护一个依赖列表，当这个对象的状态发生改变时，会自动通知所有依赖于它的对象。

让页面动起来

发表于 2018-07-24 | 分类于 UI相关

transition动画

随着css3的广泛使用，或多或少都会用到transition的属性transition-timing-function(根据时间的推进改变属性值的变化速率)来做动画效果。而这个属性本身带有6个值可供选择：

  1. ease –(逐渐变慢) 默认值
  2. linear –(匀速)
  3. ease-in –(加速)
  4. ease-out –(减速)
  5. ease-in-out –(加速然后减速)
  6. cubic-bezier --(自定义时间曲线)

上面前5个都有相应的贝塞尔曲线，如ease对应的贝塞尔曲线是cubic-bezier(0.25, 0.1, 0.25, 1.0)。然而，正是因为有5个现成的效果，导致cubic-bezier经常被忽视。这里主要了解怎么用cubic-bezier。

了解网页Icon的用法

发表于 2018-07-18 | 分类于 UI相关

前言

在如今的Web页面中，随便打开一个网站，其中图片所占的比例是非常高的。这些图片不仅包含banner或者产品图，还有用于提示作用的小图标等。当然啦，有了这些图片的引导，页面的可读性和美观性都有所提升。可是每加载一个图片，都会产生一次http请求，影响网站的响应速度。
为了减少http的请求次数，前端er总结出许多方法，从最初的css雪碧图 -> css3制作图标 -> font字体 -> svg矢量图，这些方法可以在不同的场景下使用。下面，我们就来一一介绍吧~

CSS雪碧图

原理：把许多小图标合成一张大图，在使用时，通过background-position属性来精确获取具体的图标位置。这样的好处是把多个请求合并成一个。缺点是这种方法不适用banner图或者产品图等大图。

这儿有个很好的在线工具，很方便制作雪碧图，而且背景位置都给你标出来了，直接copy就好。
